## Step One: Configure Github

<Note>
  If you're going to use the experimental branch you can skip this first step.
</Note>

### Make a New Branch

1. Go to your Github Desktop App and go to Current Branches in the top bar and click on it to reveal the option to create a new branch. Create a New Branch. Preferably choose to create a branch out of the experimental branch.
2. Publish the Branch to Github.
3. If you're using VSCode it should already be on the new branch now that you're using it. Keep it there.
4. Go to your [github.com](http://github.com) repositories. Create a New GitHub Repository. Call it whatever you want, but for recognition reasons, something related to your Shopify build. Make sure its private.
5. Next you're going to create add a remote repository. The code below goes over this.

````powershell expandable lines wrap
The git remote add command takes two arguments:

    A remote name, for example, origin
    A remote URL, for example, https://github.com/OWNER/REPOSITORY.git

For example:

$ git remote add origin https://github.com/OWNER/REPOSITORY.git
# Set a new remote

$ git remote -v
# Verify new remote
> origin  https://github.com/OWNER/REPOSITORY.git (fetch)
> origin  https://github.com/OWNER/REPOSITORY.git (push)



git remote add garcia git@github.com:danielgarcia/garcia-repo.git
#              ↑           ↑              ↑              ↑
#           nickname    SSH protocol   github username  repo name



git push (chosen-partner-repo) (your-local-branch):(to-their-localbranch) --- This Command pushes to desired remote repo from our desired branch to the desired branch of theirs

for example:

git push PartnerRepo experimental:main

```

> [!WARNING]
> MAKE SURE TO ADD YOURSELF AS A COLLABORATOR ON THEIR GITHUB REPO SO YOU CAN PUSH THE CHANGES.

<code>
# Verify it's there
git remote -v
</code>

<code>
# Remove Remote Repo
git remote remove (repo name)
</code>
````

6. These are the commands you should be using in sequence.

```powershell lines wrap
git remote add <REMOTE NAME> <REMOTE URL>
git push -u <REMOTE NAME> <CHOSEN BRANCH TO SET UPSTREAM FROM>
#The upstream here basically allow us to just push changes from our branch and our remote branch would also get the changes pushed as well.
```

7. After this let's push this repository to Hydrogen. Go to Hydrogen App → Storefront Settings → Oxygen Deployments
8. Connect your Github account and your repository.
9. Create a new Oxygen Deplyoment token. It is in the Oxygen Deployments Section
10. Go to your Github Repo that's connected to your Hydrogen project then → Settings → Secrets & Variables → Actions.
11. Now use the token value that you created and place it in the Repository Secrets. Call it `SHOPIFY_HYDROGEN_DEPLOYMENT_TOKEN`

<Danger>
  MAKE SURE YOUR ENVIRONMENT VARIABLES IN HYDROGEN ARE SET OR NOTHING WILL WORK IN PROD OR PREVIEW.

  You need these:

  **SANITY\_API\_TOKEN**: This is the viewer role permission token that we created in the API settings in our Sanity project.

  **SANITY\_PROJECT\_ID**: Get it from the Sanity Project Dashboard
</Danger>

## Step 2: Configure Production Domains for Store & Checkout

You need to setup both the Checkout Domain and the Production Domain

Documentation on this is here : [https://help.shopify.com/en/manual/domains/add-a-domain](https://help.shopify.com/en/manual/domains/add-a-domain)

1. Go to Admin Settings → Domains
2. Next click on Connect Existing on the top right and enter the domain you wish to use. For checkout domains use the checkout.\[domain\] for the URL.
3. Change the domain types to be the primary ones.

## Step 3: Setup Sanity Studio

1. Go to your /studio and in your terminal use

```powershell
npx sanity undeploy
```

3. If that doesn't work then delete the value of `sanityAppId `in your `QuickHardcodes.ts`
4. Then use this command to deploy a new studio.

```powershell
npx sanity deploy
```

5. Save the App Id that is created and shown in the terminal after deployment. Put it in the your `QuickHardcodes.ts` file

### Sanity Studio Hosted

1. After completing the previous section your new studio should appear on your Studios & Applications tab in your Sanity Organization Dashboard.
2. Go [https://www.sanity.io/manage](https://www.sanity.io/manage) → Choose Your Organization → Open Dashboard → Studios & Apps
3. Get the URL of your new studio and replace the studioDomain hardcode in your `QuickHardcodes.ts` file
4. Choose the organization you're working on and **open the organization dashboard**.
5. Go to **Studio & Apps** in the sidebar.
6. Go [https://www.sanity.io/manage](https://www.sanity.io/manage)
7. Choose the project you're working on.
8. Go to the Studio tab in your project.
9. Add a Studio
